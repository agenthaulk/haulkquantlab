## 核心问题

量化金融面临的挑战:
1. **高维性**: 多因子、多资产
2. **非平稳性**: 市场分布漂移
3. **波动性**: 尾部风险

现有方案的局限:
- ❌ 自动化程度低，依赖人工
- ❌ 可解释性差，LLM 直接输出信号易幻觉
- ❌ 因子和模型优化割裂，缺乏协同

## RD-Agent(Q) 架构

### 整体流程
```
┌─────────────────────────────────────────────────────────┐
│                    RESEARCH 阶段                        │
│  Specification Unit → Synthesis Unit                    │
│  (场景定义)           (假设生成 + 知识森林)              │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                   DEVELOPMENT 阶段                      │
│  Implementation Unit → Validation Unit → Analysis Unit  │
│  (Co-STEER 代码生成)   (Qlib 回测)     (结果评估+调度)   │
└─────────────────────────────────────────────────────────┘
                          ↑
                    反馈闭环 ←───────────────┘
```

### 五大模块详解

#### 1️⃣ Specification Unit
**作用**: 定义统一接口，屏蔽底层细节

形式化: S = (B, D, F, M)
- B: 背景知识/先验假设
- D: 数据接口 (市场数据格式)
- F: 输出格式 (因子张量/收益预测)
- M: 执行环境 (Qlib 回测)

**约束**: ∀x ∈ D, f_θ(x) ∈ F 且在 M 中可执行

#### 2️⃣ Synthesis Unit
**作用**: 模拟人类研究员，生成新假设

核心机制:
- 历史实验轨迹: e_t = {h_t, f_t} (假设 + 反馈)
- SOTA 集合维护
- 条件子集提取:
  ```
  F(a)_t = {f^a_i ∈ F_t | a = a_t ∨ e_i ∈ SOTA(a)}
  H(a)_t = {h^a_i ∈ H_t | a = a_t ∨ h_i ∈ SOTA(a)}
  ```
- 生成函数: h_(t+1) = G(H(a)_t, F(a)_t)

**知识森林**:
- 成功 → 增加复杂度/范围
- 失败 → 结构调整/引入新变量
- 保持多样性 + 渐进优化

#### 3️⃣ Implementation Unit (Co-STEER)
**核心创新**: 面向数据中心的代码生成智能体

**任务调度**:
- DAG 表示依赖关系: G = (V, E)
- 拓扑排序: π_S = (t^(1), ..., t^(n))
- 自适应调整: 失败多次的任务降优先级

**知识演化**:
```
执行轨迹 + 反馈 → 知识库
           ↓
相似错误检索 → 修正提示
           ↓
迁移学习 → 跨任务复用
```

**示例错误处理**:
```
错误: 'MultiIndex' object has no attribute 'date'
检索: 相似代码使用 get_level_values()
修正: 替换为正确 API
```

#### 4️⃣ Validation Unit
**作用**: 在真实市场回测

- 基于 Qlib 框架
- 标准化评估指标
- 统一性能度量

#### 5️⃣ Analysis Unit
**作用**: 评估结果 + 选择下一方向

**多臂老虎机调度**:
- 动作空间: a_t ∈ {factor, model}
- 探索 vs 利用平衡
- 自适应方向选择

## 关键技术

### Co-STEER 智能体
```
┌─────────────────┐
│  候选任务队列   │
└────────┬────────┘
         ↓
┌─────────────────┐     ┌─────────────────┐
│  调度模块       │ ←── │  执行反馈       │
│  (复杂度/依赖)  │     └─────────────────┘
└────────┬────────┘
         ↓
┌─────────────────┐
│  实现模块       │
│  (代码生成)     │
└────────┬────────┘
         ↓
┌─────────────────┐
│  知识库         │ ←── 粒度化设计，可迁移
└─────────────────┘
```

### 因子-模型协同优化
**策略**: 交替优化
1. 固定模型 → 优化因子
2. 固定因子 → 优化模型
3. 循环迭代

**优势**:
- 70% 更少因子
- 2× 年化收益
- 更好的鲁棒性

## 实验结果

### 数据集
- 真实股票市场数据
- 时间跨度: 多年历史数据

### 对比基线
1. 经典因子库 (Fama-French 等)
2. 深度时序模型 (LSTM, GRU, Transformer)
3. SOTA 时序预测模型

### 性能指标
| 指标 | RD-Agent(Q) | 基线 |
|------|-------------|------|
| 年化收益 | **2×** | 1× |
| 因子数量 | **30%** | 100% |
| 成本 | **<$10** | - |
| 可解释性 | ✅ 代码可审计 | ❌ 黑盒 |

### 消融实验
- Co-STEER vs 普通代码生成: +35% 准确率
- 知识森林 vs 单假设: +28% 多样性
- MAB 调度 vs 随机: +42% 效率
